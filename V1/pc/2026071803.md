## 优化pinia配置

之前的配置src\stores\index.ts

```ts
import { defineStore } from 'pinia'

export const useSidStore = defineStore('sid', {
  //存数据的地方
  state() {
    return {
      sid: '',
    }
  },
  //actions里面放置的是一个一个的方法，用于响应组件中的“动作”
  actions: {
    // 存放唯一标识
    setSid(value: string) {
      this.sid = value
    },
  },

  getters: {},
})
```

- 命名不合理，并且只有一个index，通常不会把所有的状态放在index，所以修改一下，

创建src\stores\auth.ts

```ts
import { defineStore } from 'pinia'

export const useAuthStore = defineStore('auth', {
  //存数据的地方
  state() {
    return {
      sid: '',
      isLogin: false,
      token: '',
    }
  },
  //actions里面放置的是一个一个的方法，用于响应组件中的“动作”
  actions: {
    // 存放唯一标识
    setSid(value: string) {
      this.sid = value
    },
  },

  getters: {},
})

```

为了后续项目需要，这里多创建几个

`src\stores\user.ts`、`src\stores\app.ts`

```ts
import { defineStore } from 'pinia'

export const useUserStore = defineStore('user', {
  state: () => ({}),

  getters: {},

  actions: {},
})

```

```ts
import { defineStore } from 'pinia'

export const useAppStore = defineStore('app', {
  state: () => ({}),

  actions: {},
})

```

然后index导入

```ts
export * from './auth'
export * from './user'
export * from './app'
```

修改之后之前的引用会报错，所以需要修改：这里全文查找`useSidStore`替换为`useAuthStore`

```ts
import { useAuthStore } from '@/stores'
sid: useAuthStore().sid,
```

这样，pinia的配置就优化好了，这样命名和划分就舒服多了

## 继续完善isLogin

pinia配置好之后，我们就需要从pinia取isLogin了，但是这里需要注意的是，如果直接这样写，会丢失响应式：

```ts
const isLogin = useAuthStore().isLogin
```

正确的写法,使用storeToRefs

```ts
const { isLogin } = storeToRefs(useAuthStore())
```

当然也可以直接使用

```ts
<template v-if="useAuthStore().isLogin">
```

## 首页接口开发

创建：src\api\content.ts

```ts
import type { getListInfo } from '@/common/interface'
import axios from '@/common/request'
import qs from 'qs'
/**
 * 读取文章列表
 * @param options 读取文章列表参数
 * @returns
 */
const getList = (options: getListInfo) => {
  return axios.get('/public/list? ' + qs.stringify(options))
}
/**
 * 本周热议
 * @returns
 */
const getTop = () => {
  return axios.get('/public/topWeek ')
}

/**
 *温馨提醒接口
 * @returns
 */
const getTips = () => {
  return axios.get('/public/tips ')
}
/**
 *友情链接
 * @returns
 */
const getLinks = () => {
  return axios.get('/public/tips ')
}

export { getList, getTips, getLinks, getTop }

```

这里使用到了qs，需要安装qs

```shell
npm install -S qs
```

除此之外还需要按照

```shell
npm install -D @types/qs
```

getListInfo这个类型需要添加：src\common\interface.ts

```ts
export interface getListInfo {
  isTop: number
  page: number
  limit?: number
  catalog?: string
  sort?: string
  status?: string
}
```
