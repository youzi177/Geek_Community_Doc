# vee-validate后端返回错误显示

本章主要实现vee-validate后端返回错误进行显示并且简单封装一下组件的函数

有这样的登录场景：用户输入一个邮箱，一个密码，一个验证码，在前端校验都通过了，起码是一个邮箱，密码是一个至少6位的，验证码至少是4位的，但是不知道用户输入的邮箱和密码是否匹配，验证码是否正确，这些都是需要后端判断然后在返回的，那返回的错误要给用户提示是哪里错误的，如验证码错，就在验证码的地方显示错误，用户名和密码不匹配就弹窗一个不匹配的信息，让用户重新输入，所以本章主要就是完成这个功能的。

## 改写之前的vee-validate

之前的Form是这样的：

```vue
<Form ref="object" v-slot="{ validate, errors }"></Form>
```

现在改为：

```vue
<Form @submit="submit" v-slot="{ errors }"></Form>
```

把按钮也改为submit：

```vue
 <button class="layui-btn" type="submit">立即登录</button>
```

这里先举例一下找回密码这个组件：

```ts
//找回密码
const submit = async (value: any, actions: any) => {
  const { setErrors } = actions
  const result = await forget({
    username: state.username,
    code: state.code,
    sid: useSidStore().sid,
  })
  //明确告知result就是HttpResponse类型
  const { code, msg } = result as HttpResponse
  if (code === 200) {
    alert(msg)
  } else if (code === 401) {
    setErrors({
      code: msg,
    })
  }
}
```

- submit会首先校验整个表单，如果有错误就会展示
- 接收两个参数：value（表单内容），actions（表单操作方法集合）
- 从actions解构出`setErrors`，这个就是重点，参数是一个对象，对象属性是表单的name，值是后端返回的信息
- 这里就是请求forget接口，返回result，从result解构出后端返回的code和msg，然后判断code是否401，是就`setErrors`
- 其他组件都是这样的思路

## 封装登录注册找回密码的函数

在登录、注册、找回密码这三个组件中都有验证码这个功能以及一些重复的data，可以把这些逻辑封装起来

创建：`src\hooks\Uselogin.ts`

```ts
import { useSidStore } from '@/stores'
import { reactive } from 'vue'
import { v4 as uuidv4 } from 'uuid'
import { getCode } from '@/api/login'
import type { HttpResponse } from '@/common/interface'
export default () => {
  //数据
  const state = reactive({
    username: '',
    name: '',
    password: '',
    repassword: '',
    code: '',
    svg: '',
  })
  //设置sid
  const setid = () => {
    let sid = ''
    //从localStorage取值sid
    if (localStorage.getItem('sid')) {
      sid = localStorage.getItem('sid') || ''
      //存到pinia
    } else {
      //没有sid就生成一个并存入localStorage
      sid = uuidv4()
      localStorage.setItem('sid', sid)
      //存到pinia
    }
    useSidStore().setSid(sid)
  }
  //发送验证码
  const _getCode = async () => {
    const sid = useSidStore().sid
    //请求验证码
    const result = (await getCode(sid)) as HttpResponse
    //解构
    const { code, data } = result
    if (code === 200) {
      state.svg = data as string
    }
  }
  return {
    state,
    setid,
    _getCode,
  }
}
```

使用：

```ts
import Uselogin from '@/hooks/Uselogin'
//封装函数
const { state, _getCode, setid } = Uselogin()

const { username, code, svg } = toRefs(state)
onMounted(() => {
  setid()
  _getCode()
})
```

- Uselogin其实`export default`一个函数，然后返回里面的函数，这样，我们在组件中代码就很干净了

## 完善登录注册接口

修改接口：src\api\login.ts

```ts
import type { ForgetInfo, LoginInfo, RegInfo } from '@/common/interface'
import axios from '@/common/request'

//验证码
const getCode = async (sid: string) => {
  return await axios.get('/public/getCaptcha', {
    params: {
      sid: sid,
    },
  })
}
//找回密码
const forget = async (option: ForgetInfo) => {
  return await axios.post('/login/forget', { ...option })
}
//登录
const login = async (option: LoginInfo) => {
  return await axios.post('/login/login', { ...option })
}
//注册
const reg = async (option: RegInfo) => {
  return await axios.post('/login/reg', { ...option })
}
export { getCode, forget, login, reg }

```

完善src\common\interface.ts

```ts
export interface ForgetInfo {
  username: string
  code: string
  sid: string
}

export interface LoginInfo {
  password: string
  username: string
  code: string
  sid: string
}
export interface RegInfo {
  password: string
  username: string
  name: string
  repassword?: string
  code: string
  sid: string
}
//后端返回的数据格式
export interface HttpResponse {
  code: number
  //unknown：需要类型判断后才能使用
  data?: unknown
  msg?: string
}
```
