# 优化vee-validate

之前的配置vee-validate并没有在提交的时候验证表单，会出现表单不通过的情况下依然能触发登录、注册、提交等事件，本章将优化这个问题

## 三种解决方法

第一种：使用 `handleSubmit`

```vue
<Form v-slot="{ handleSubmit, errors }">
  <form @submit="handleSubmit(login)">
    <button class="layui-btn" type="submit">
      立即登录
    </button>
  </form>
</Form>
```

使用

```ts
const login = async (values) => {
  console.log('校验通过', values)
  console.log('登录成功')
}
```

这种方式需要再里面套一层form，然后通过submit事件校验登录，这个结构可以使用，但是由于之前不是这样写的，这里就不用这种方式。

第二种：手动调用 `validate`

```vue
<Form ref="object" v-slot="{ validate, errors }">
  <button 
    class="layui-btn" 
    type="button"
    @click="login(validate)"
  >
    立即登录
  </button>
</Form>
```

使用：

```ts
const login = async (validate) => {
  const { valid } = await validate()

  if (!valid) {
    console.log('校验失败')
    return
  }

  console.log('登录成功')
}
```

这是项目使用的方式，不破坏原来结构，通过参数validate校验，需要注意 `type="button"`不能是`type="submit"`，之前是没有写type的，模式就是submit

第三种：使用 `Form` 的 submit 事件

```ts
<Form @submit="login" v-slot="{ errors }">
  <button class="layui-btn" type="submit">
    立即登录
  </button>
</Form>
```

使用：

```ts
const login = (values) => {
  console.log('校验通过', values)
}
```

这是最简单的方式，就是通过submit事件执行，这个values就是表单的数据，注意`type="submit"`
