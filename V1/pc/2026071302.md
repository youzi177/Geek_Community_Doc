# 图片验证码校验和存储

本章实现图片验证码校验和存储，简单来说，前端获取验证码的时候带上一个标识（UUID），并存储在`localStorage`，后端拿到这个标识之后，和图片验证码绑定，存入到`redis`中并且设置过期时间，等到登录注册或者其他需要校验的时候在从`redis`取值比对。

UUID存在`localStorage`不就一直不变吗，是的，就是需要这个key值不变，但是这个是相对于同一个设备来说，不同设备标识都不一样的，如果一台设备每次请求都是一个新的key值，那如果疯狂点击验证码，服务器就崩了，redis是存在内存中的，数据库也会突然暴涨，所以UUID存在`localStorage`是最稳妥的

## 安装UUID

```
npm install uuid
```

## 使用UUID

src\pages\login.vue

```ts
import { v4 as uuidv4 } from 'uuid'
import type { HttpResponse } from '@/common/interface'
const _getCode = async () => {
  let sid = ''
  //从localStorage取值sid
  if (localStorage.getItem('sid')) {
    sid = localStorage.getItem('sid') || ''
  } else {
    //没有sid就生成一个并存入localStorage
    sid = uuidv4()
    localStorage.setItem('sid', sid)
  }
  //请求验证码
  const result = (await getCode(sid)) as HttpResponse
  //解构
  const { code, data } = result
  if (code === 200) {
    state.svg = data as string
  }
}
```

src\api\login.ts

```ts
//验证码
const getCode = async (sid: string) => {
  return await axios.get('/public/getCaptcha', {
    params: {
      sid: sid,
    },
  })
}
```

forget和reg组件也是一样的。

- 就是从`localStorage`取值`sid`，没有取到就就生成一个`uuid`并存入`localStorage`，然后发送请求的时候带上sid

至此，本章结束，接下来是后端那这个`sid`，并存储到`redis`中，设置验证码过期时间

## 存放到pinia

修改src\stores\index.ts

```ts
import { defineStore } from 'pinia'

export const useSidStore = defineStore('Sid', {
  //actions里面放置的是一个一个的方法，用于响应组件中的“动作”
  actions: {
    // 存放唯一标识
    setSid(value: string) {
      this.sid = value
    },
  },
  //存数据的地方
  state() {
    return {
      sid: '',
    }
  },
  getters: {},
})

```

使用

```ts
import { useSidStore } from '@/stores'
//验证码
const _getCode = async () => {
  let sid = ''
  //从localStorage取值sid
  if (localStorage.getItem('sid')) {

  } else {
    //没有sid就生成一个并存入localStorage

    localStorage.setItem('sid', sid)
  }
  //存到pinia
  useSidStore().setSid(sid)
  //请求验证码
  const result = (await getCode(sid)) as HttpResponse
  //解构
  const { code, data } = result
  if (code === 200) {
    state.svg = data as string
  }
  const a = useSidStore()
}
```
