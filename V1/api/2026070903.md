# 第一个restful应用

本章主要实现一个restful基础应用，不涉及项目逻辑，使用webpac打包

koa官网：https://koa.node.org.cn/

## 项目初始化

创建项目目录,初始化package.json

```
npm init -y
```

安装koa

```
npm i koa
```

创建index.js

```js
const Koa = require('koa');
const app = new Koa();

app.use(async ctx => {
  ctx.body = 'Hello World';
});

app.listen(3000);
```

运行

```shell
node index.js
```

打开浏览器：http://localhost:3000/ ，就可以看到Hello World，说明koa正常启动了，这个脚本其实很简单，就是引入koa，实例化一个koa，use一个中间件，上下文返回一个Hello World，然后监听3000端口。

## 添加路由

koa本身是没有路由，需要通过中间件实现；

安装

```shell
npm i koa-router -S
```

使用：

```js
const Koa = require('koa');
const Router = require('koa-router');
const app = new Koa();
const router = new Router();

router.prefix('/v1');

router.get('/', (ctx) => {
  ctx.body = 'Hello World';
});
router.get('/api', (ctx) => {
  ctx.body = 'Hello api';
});

app.use(router.routes()).use(router.allowedMethods());
app.listen(3000);

```

访问：http://localhost:3000/v1 可以看到Hello World，访问：http://localhost:3000/v1/api 可以看到Hello api

`router.prefix('/v1');`就是给路由加上前缀

## 添加协议解析和跨域处理中间件

安装koa-body和@koa/cors

```shell
npm i koa-body @koa/cors -S
```

使用：

```js
const Koa = require('koa');
const Router = require('koa-router');
const { koaBody } = require('koa-body');
const cors = require('@koa/cors');
const app = new Koa();
const router = new Router();

router.get('/', (ctx) => {
  ctx.body = 'Hello World';
});
router.get('/api', (ctx) => {
  // 获取get参数
  const params = ctx.request.query
  console.log(params);
  ctx.body = {
    data: 'this is data ',
    code: 200,
    params:params
  };
});
router.post('/post', (ctx) => {
  // 获取post参数
  let { body } = ctx.request;
  console.log(body);
  ctx.body = {
    ...body,
  };
});

app.use(koaBody());
app.use(cors());
app.use(router.routes()).use(router.allowedMethods());
app.listen(3000);
```

## 工程化目录

把所有代码放在一起，这显然不优雅，接下来创建工程化的目录

```
├─config --------------- // webpac配置
├─package-lock.json 
├─package.json 
├─public --------------- // 静态资源文件
└─src 
  ├─api ---------------- // 业务逻辑
  ├─index.js ----------- // koa入口
  └─router ------------- // 路由
    ├─modules ---------- // 业务路由,方便处理路由的prefix前缀
    └─routes.js -------- // 集合路由
```

- 把所有代码都放在src目录下，主要分为3层，index->router->api

## webpack配置

接下来把项目打包成一个js文件

安装必要的依赖：

```shell
npm i webpack webpack-cli webpack-node-externals @babel/core babel-loader @babel/preset-env cross-env -D
```

- webpack webpack-cli 是webpack 的核心；

- webpack-node-externals 是 排除掉node_modules，不做处理

- @babel/core 是babel核心，使用babel-loader 就必须安装

- babel-loader 是webpack的loader

- @babel/preset-env：根据目标环境转换 JS 语法

- cross-env设置环境变量， 是一个用来跨平台设置环境变量的 npm 工具，解决的是 Windows 和 Linux/macOS 设置环境变量写法不一致的问题：

  ```json
  {
    "scripts": {
      "build": "cross-env NODE_ENV=production webpack --config config/webpack.config.js",
      "dev": "cross-env NODE_ENV=development node src/index.js"
    }
  }
  ```

  `cross-env NODE_ENV=production`设置当前的环境为生产模式；`cross-env NODE_ENV=development`开发模式，这个命令可以解决后续webpack分为生产和开发的配置的时候使用

在config文件夹下创建两个文件：webpack.config.js和utils.js

```js
//webpack.config.js
const path = require('path');
const utils = require('./utils');
const nodeExclude = require('webpack-node-externals');
const webpackconfig = {
  mode: 'development', //开发模式
  target: 'node', //运行环境
  entry: {
    server: path.join(utils.APP_PATH, 'index.js'),
  },
  output: {
    path: utils.DIST_PATH, //输出的路径，需要绝对路径， //输出的路径，需要绝对路径，
    filename: `[name].bundle.js`, //输出的名字
    clean: true, //清除dist
  },
  devtool: 'eval-source-map', //开启source-map，错误的时候可以定位源码
  module: {
    rules: [
      // 使用babel-loader，支持ES6语法，排除node_modules文件
      {
        test: /\.(js|jsx)$/,
        use: {
          loader: 'babel-loader',
        },
        exclude: [path.join(__dirname, './node_modules')], //排除node_modules
      },
    ],
  },
  externals: [nodeExclude()], //排除node_modules
};
module.exports = webpackconfig;
```

```js
//utils.js
const path = require('path');

exports.resolve = function resolve(dir) {
  return path.join(__dirname, '..', dir);
};
exports.APP_PATH = exports.resolve('src');
exports.DIST_PATH = exports.resolve('dist');

```

根目录创建.babelrc：

使用 `@babel/preset-env` 根据当前 Node 环境自动决定需要转换哪些 JavaScript 语法。

`@babel/preset-env` 是一个智能预设，它会根据目标环境转换代码。

`"node": "current"`：目标运行环境是当前正在运行 Babel 的 Node 版本。

```js
{
  "presets": [
    [
      "@babel/preset-env",
      {
        "targets": {
          "node": "current"
        }
      }
    ]
  ]
}

```

运行打包命令：

```shell
npx webpack --config config/webpack.config.js
```

可以看到根目录下有dist目录及server.bundle.js 运行它

```shell
node dist/server.bundle.js
```

但是之前配置的nodemon就不能用了，也不是不能用，而是不在监听原来的index代码，而是需要监听webpack打包后的代码

webpack提供了一个watch功能，可以监控入口文件及其依赖的变化，文件修改后自动重新打包，但是只是打包，并不会开启服务端口，所以需要配合nodemon，下面给出最简单，最直接，最不绕弯子的答案

安装npm-run-all

```shell
npm install -D npm-run-all
```

配置

```json
"scripts": {
    "start": "nodemon src/index.js",
    "watch": "webpack --config config/webpack.config.js --watch",
    "debug": "nodemon --inspect=9229 ./dist/server.bundle.js",
    "dev": "npm-run-all -p watch debug"
},
```

运行：

```shell
npm run dev
```

## 优化webpack配置

上面的配置能基本使用，但是还是不够，缺少线上环境打包，之前安装的cross-env也没有用，`mode: 'development'`也是写死的，接下来优化一下webpack配置

简单来说就是创建3个配置文件，webpack.config.base.js、webpack.config.dev.js和webpack.config.prod.js，名字就可以看出来它们的用意，一个基础的配置 ，一个开发模式，一个生产模式，然后搭配`cross-env`，线上打包一个命令，开发一个命令

安装：webpack merge和terser-webpack-plugin

webpack merge用来合并配置，terser-webpack-plugin用来压缩代码和自定义压缩规则（webpack5默认已经带 Terser）

```shell
npm i webpack-merge terser-webpack-plugin -D
```

`config\webpack.config.base.js`

```js
const path = require('path');
const utils = require('./utils');
const nodeExclude = require('webpack-node-externals');
const webpack = require('webpack');
const webpackconfig = {
  target: 'node', //运行环境
  entry: {
    server: path.join(utils.APP_PATH, 'index.js'),
  },
  output: {
    path: utils.DIST_PATH, //输出的路径，需要绝对路径， //输出的路径，需要绝对路径，
    filename: `[name].bundle.js`, //输出的名字
    clean: true, //清除dist
  },
  // 使用webpack5之后，webpack会从配置文件的mode中自动为process.env.NODE_ENV赋值，而取的值，就是该配置文件的mode属性。如果没有值，则会默认返回“production”。
  // 让webpack不会自动读取配置文件中的mode给process.env.NODE_ENV赋值。
  // 这样process.env.NODE_ENV就只是被我们自定义的文件赋值，就不会冲突了。
  optimization: {
    nodeEnv: false,
  },
  devtool: 'eval-source-map', //开启source-map，错误的时候可以定位源码
  module: {
    rules: [
      // 使用babel-loader，支持ES6语法，排除node_modules文件
      {
        test: /\.(js|jsx)$/,
        use: {
          loader: 'babel-loader',
        },
        exclude: [path.join(__dirname, './node_modules')], //排除node_modules
      },
    ],
  },
  externals: [nodeExclude()], //排除node_modules
  //因为前面的optimization为false，所以这里需要设置EnvironmentPlugin
  //作用：把Node进程里的环境变量，注入到 webpack 打包后的代码中，让代码里可以使用 process.env.NODE_ENV
  //本质是DefinePlugin 的一个方便封装
  //让cross-env NODE_ENV=development控制，
  plugins: [
    new webpack.EnvironmentPlugin(['NODE_ENV']), // 可以直接使用 environmentPlugin
  ],
};

module.exports = webpackconfig;

```

`config\webpack.config.dev.js`

```js
const webpackMerge = require('webpack-merge')
const baseWbpackConfig = require('./webpack.config.base')
const webpackConfig = webpackMerge.merge(baseWbpackConfig, {
  devtool: 'eval-source-map', //开启source-map
  mode: 'development', //开发模式
  stats:{children:false}//不显示子编译（child compilations）的构建信息

module.exports = webpackConfig;
```

```js
});
```

## 优化代码

本章进入尾声，项目基本已经搭建好骨架，但是之前的代码需要重新改写一下：

> 注意：package.json中不能有"type": "commonjs",否则报错。

改写之前先安装两个中间件：koa-compress是生产模式下压缩中间件，koa-compose是整合中间件

```shell
npm i koa-compress koa-compose -S
```

src\index.js

```js
import Koa from 'koa';
import router from './router/routes';
import { koaBody } from 'koa-body';
import cors from '@koa/cors';
import json from 'koa-json';
import helmet from 'koa-helmet';
import statics from 'koa-static';
import path from 'path';
import compose from 'koa-compose';
import compress from 'koa-compress';
const app = new Koa();

//process.env.NODE_ENV 不等于 'production'那就是true，就是开发模式，否则生产模式
const isDevMode = process.env.NODE_ENV !== 'production';
// 开发模式为3000端口，生产模式为12006
const port = !isDevMode ? '12006' : '3000';

// 中间件
// compose集合中间件
const middleware = compose([
  koaBody(),
  cors(),
  json(),
  helmet(),
  statics(path.join(__dirname, '../public')),
]);
app.use(middleware);
app.use(router());
// 生产模式压缩中间件
if (!isDevMode) {
  app.use(compress());
}
// 监听3000端口
app.listen(port, () => {
  console.log(`The server is runing at: ${port}`);
});

```
