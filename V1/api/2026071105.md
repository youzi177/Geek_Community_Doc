# 服务器环境搭建

本章将对服务器进行基础的搭建，至于为什么突然中间来一个服务器环境搭建，不应该在最前面或者最后上线才考虑吗，其实前面的这是一个基础热身环境，如果一上来就各种环境搭建，容易打击开发项目的热情，为什么不放到最后是因为项目写到这里，不得不需要数据库了。

除了服务器基本的配置，还会搭建docker和docker-compose的环境，使用docker安装几个工具，以及配置`https`、`nginx`，让`nginx`容器代理其他容器，从而实现`https`的访问，这些安装和配置都是为了未来项目的自动化集成部署做准备。本章虽然与业务逻辑无关，但是也是绕不过的，因为后期的数据库是通过docker安装的

这里直接用最简单最直白的话推荐服务器：临时开发使用推荐阿里云的99元一年2核2G的服务器：活动地址：https://www.aliyun.com/daily-act/ecs/99program

这是个人认为最划算的一种，对比其他云商家，价格虽然低，但是有的需要抢，还是限制流量的，对比阿里云性价比还是低了一点，至少不用担心流量问题

## 服务器连接配置

首先：修改服务器默认的22端口，22端口太容易被攻击，修改为自己想要的端口

1. 连接：

   ```shell
   ssh root@you ip
   ```

2. 修改端口，编辑`sshd_config`文件：

   ```shell
   vi /etc/ssh/sshd_config
   ```

3. 输入 i 编辑，把#去掉，把22改为想要的端口，最后:wq保存：

   ```bash
   Port 10024
   ```

4. 用编辑器打开 `/lib/systemd/system/ssh.socket` 文件

   ```shell
   sudo vim /lib/systemd/system/ssh.socket
   ```

5. 将 `ListenStream=22` 中的 22 端口号修改为你刚才设置的端口号

   ```bash
   ListenStream=10024
   ```

6. 重启服务

   ```shell
   sudo systemctl daemon-reload
   ```

   ```shell
   sudo systemctl restart ssh.socket
   ```

7. 测试登录：

   ```shell
   ssh -p 10024 root@your ip
   ```

## 服务器密钥登录

每次登录都需要密码，显然很麻烦，这里就推荐使用密钥登录

1. 在本地电脑生成SSH Key，一直回车就可以：

   ```shell
   ssh-keygen
   ```

2. 在本地路径C:\Users\你的用户\.ssh下创建`config`文件：

   ```
   Host Test
       Port #端口号
       HostName #IP地址
       User root
       IdentityFile ~/.ssh/id_rsa #本地id_rsa文件
       IdentitiesOnly yes
   ```

3. 服务器`cd ~/.ssh/`目录，（一定要CD到这个目录，切勿直接在根目录直接使用下面命令，会导致这个文件不生效）编辑文件

   ```shell
   vi authorized_keys
   ```

4. 到本地C:\Users\你的用户名\.ssh 找到公钥文件，把本地电脑的公钥文件`id_rsa.pub`复制进去，`:wq`保存

5. 测试：

   ```shell
   ssh Test
   ```

6. 如果之前连接过服务器，格式化后再连接可能报错，可以先删除本地电脑know_hosts和know_hosts_old两个文件，这两个文件是历史连接的记录

## Docker和dockercompose的安装

阿里云安装Docke文档：https://help.aliyun.com/zh/ecs/use-cases/install-and-use-docker-on-a-linux-ecs-instance

如何安装可以参考上边的文档，这里做一个备份，因为随着时间推移，系统更新，docker更新，可能有些存在不适配的情况，这里使用`Ubuntu 26.04 64位` 系统版本

1. 更新包管理工具

   ```shell
   sudo apt-get update
   ```

   ```shell
   sudo apt-get -y install apt-transport-https ca-certificates curl software-properties-common
   ```

2. 添加Docker软件包源（使用keyrings方式管理GPG密钥）

   ```shell
   sudo install -m 0755 -d /etc/apt/keyrings
   ```

   ```shell
   curl -fsSL http://mirrors.cloud.aliyuncs.com/docker-ce/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
   ```

   ```shell
   sudo chmod a+r /etc/apt/keyrings/docker.gpg
   ```

3. Docker官方源暂未提供Ubuntu 26对应仓库，此处统一使用noble（Ubuntu 24.04 LTS）源，兼容Ubuntu 26等较新版本

   ```shell
   ARCH=$(dpkg --print-architecture)
   ```

   ```shell
   DISTRO="noble"
   ```

   ```shell
   sudo tee /etc/apt/sources.list.d/docker.list > /dev/null <<EOF
   deb [arch=${ARCH} signed-by=/etc/apt/keyrings/docker.gpg] http://mirrors.cloud.aliyuncs.com/docker-ce/linux/ubuntu ${DISTRO} stable
   EOF
   ```

   ```shell
   sudo apt-get update
   ```

4. 安装Docker社区版本，容器运行时containerd.io，以及Docker构建和Compose插件

   ```shell
   sudo apt-get -y install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
   ```

5. 启动Docker并设置开机自启

   ```shell
   #启动Docker
   sudo systemctl start docker
   ```

   ```shell
   #设置Docker守护进程在系统启动时自动启动
   sudo systemctl enable docker
   ```

6. 测试：

   ```shell
   docker -v
   ```

   ```shell
   docker compose version
   ```

## 安装acme

acme安装如果是国内服务器可能会出现安装非常缓慢的情况或者失败，这里一样提供两种安装方式：一种是国内服务器安装，另外一种是国外服务器的安装，两者区别在于安装的速度上。

浏览器访问https://get.acme.sh/ 可以看到最后一句话：

```
#!/usr/bin/env sh #https://github.com/acmesh-official/get.acme.sh _exists() { cmd="$1" if [ -z "$cmd" ] ; then echo "Usage: _exists cmd" return 1 fi if type command >/dev/null 2>&1 ; then command -v $cmd >/dev/null 2>&1 else type $cmd >/dev/null 2>&1 fi ret="$?" return $ret } if [ -z "$BRANCH" ]; then BRANCH="master" fi #format "email=my@example.com" _email="$1" if [ "$_email" ]; then shift _email="--$(echo "$_email" | tr '=' ' ')" fi _url="https://raw.githubusercontent.com/acmesh-official/acme.sh/$BRANCH/acme.sh" _get="" if _exists curl && [ "${ACME_USE_WGET:-0}" = "0" ]; then _get="curl -L" elif _exists wget ; then _get="wget -O -" else echo "Sorry, you must have curl or wget installed first." echo "Please install either of them and try again." exit 1 fi if ! $_get "$_url" | sh -s -- --install-online $_email "$@"; then echo "Install error" echo "中国大陆用户请参考:" echo "https://github.com/acmesh-official/acme.sh/wiki/Install-in-China" fi
```

访问：https://github.com/acmesh-official/acme.sh/wiki/Install-in-China 就可以看到安装步骤：

```shell
git clone https://gitee.com/neilpang/acme.sh.git
cd acme.sh
./acme.sh --install -m my@example.com
```

其实就是去拉取gitee的仓库代码，然后在目录下使用。如果服务器没有git命令，就安装git，

```shell
sudo yum install -y git
```

安装：

```shell
cd acme.sh
```

```shell
./acme.sh --install
```

测试：

```shell
acme.sh -h
```

国外服务器：

```
curl https://get.acme.sh | sh -s email="xxxx@sss.com"
```

这个命令安装acme并且创建了一个 `cronjob`, 每天 `0:00` 点自动检测所有的证书, 如果快过期了, 需要更新, 则会自动更新证书.

## 安装Nginx

在home目录下创建文件夹`keys`和`nginx`，在`nginx`目录下创建`conf.d`文件夹。`home/nginx` 是用来存放`nginx`容器的文件；`home/nginx/conf.d`是用来存放带来其他容器的`nginx conf.d`文件；`/home/keys`，存放`Https`证书文件

1. 创建随机的`https`证书密钥：

   ```shell
   openssl dhparam -out /home/keys/dhparam.pem 2048
   ```

2. 创建容器的局域网段：

   ```shell
   docker network create https
   ```

   辅助命令：

   ```shell
   #这是查询network有哪些网段
   docker network ls
   #这是查询network里面https网段里面有哪些容器加入进来
   docker network inspect https
   ```

3. 配置`nginx.conf`文件,并上传到服务器`home/nginx`文件夹下，这是一个通用的文件配置，正常无需修改

   ```nginx
   user nginx;
   worker_processes auto;
   pid /run/nginx.pid;
   worker_rlimit_nofile 65535;
   events {
     # 设置事件驱动模型，是内核2.6以上支持
     use epoll;
     worker_connections 65535;
     accept_mutex off;
     multi_accept off;
   }
   
   http {
     # Basic Settings
     sendfile on;
     tcp_nopush on;
     tcp_nodelay on;
     send_timeout 120;
     keepalive_timeout 300;
     client_body_timeout 300;
     client_header_timeout 120;
   
     proxy_read_timeout 300;
     proxy_send_timeout 300;
     #tcp_nopush on;
     types_hash_max_size 4096;
     client_header_buffer_size 16m;
     client_max_body_size 4096m;
   
     include /etc/nginx/mime.types;
     include /etc/nginx/conf.d/*.conf;
     # include /usr/share/nginx/modules/*.conf;
   
     default_type application/octet-stream;
     # Logging Settings
     access_log /var/log/nginx/access.log;
     error_log /var/log/nginx/error.log;
     log_format main '$remote_addr - $remote_user [$time_local] "$request" '
     '$status $body_bytes_sent "$http_referer" '
     '"$http_user_agent" "$http_x_forwarded_for"';
     # 开启gzip
     gzip on;
     # 启用gzip压缩的最小文件，小于设置值的文件将不会压缩
     gzip_min_length 1k;
     # gzip 压缩级别，1-10，数字越大压缩的越好，也越占用CPU时间，后面会有详细说明
     gzip_comp_level 2;
     # 进行压缩的文件类型。javascript有多种形式。其中的值可以在 mime.types 文件中找到。
     gzip_types text/plain application/javascript application/x-javascript text/css application/xml text/javascript application/x-httpd-php image/jpeg image/gif image/png font/ttf font/otf image/svg+xml;
     # 是否在http header中添加Vary: Accept-Encoding，建议开启
     gzip_vary on;
     # 禁用IE 6 gzip
     gzip_disable "MSIE [1-6]\.";
   }
   ```

4. 编写`docker-compose`文件，安装`Nginx`容器

   ```yml
   services:
     web:
       image: nginx:latest
       container_name: "some-nginx"
       restart: always
       volumes:
         - /home/nginx/nginx.conf:/etc/nginx/nginx.conf
         - /home/nginx/conf.d:/etc/nginx/conf.d
         - /home/keys:/home/keys
         # blog
         # - /home/blog:/var/www
       ports:
         - "80:80"
         - "443:443"
   
   # docker network create https
   networks:
     default:
       name: https
       external: true
   ```

   创建了一个`some-nginx`的容器，把容器文件`nginx.conf`、`conf.d`和keys文件夹挂载出来。暴露端口是80和443；把容器加入到`https`的网段里面，让这个网段里面的容器可以被`nginx`容器代理到。

5. 把这个文件上传到服务器的`home/nginx`文件夹下，然后执行安装

   ```shell
   docker compose up -d
   ```

6. 把已经申请好的证书复制到你指定的位置，并设置续期后的自动部署命令，记得域名替换成自己的

   ```shell
   acme.sh --install-cert -d example.top \
   --key-file       /home/keys/key.pem  \
   --fullchain-file /home/keys/cert.pem \
   --reloadcmd     "docker restart some-nginx"
   ```

7. 测试，看nginx有没有正常运行

   ```
   docker ps
   ```

## 安装showdoc

上面已经做了大量的配置，这里就使用一个小示例来体验一下，`showdoc`也可以编写后续数据库和接口文档

home目录下创建showdoc文件夹，上传以下yml配置文件

`docker-compose.yml`

```yml
services:
  # ref: https://www.showdoc.cc/help?page_id=65610
  showdoc:
    image: xd2idwf5.mirror.aliyuncs.com/star7th/showdoc
    container_name: showdoc
    restart: always
    volumes:
      # chmod -R 777 /home/showdoc/html
      - /home/showdoc/html:/var/www/html/
    # ports:
    #   - "1111:80"
networks:
  default:
    name: https
    external: true

```

这个ports可以自己选择是否开放，但是个人不建议，因为使用域名https访问，不是ip加端口访问，所以没必要

安装容器，cd到showdoc这个文件夹，然后安装

```
docker compose up -d
```

在`home/nginx/conf.d`目录下上传以下`showdoc.conf`文件

```nginx
upstream show-server {
    server showdoc:80;
}
server {
    listen 80;
    server_name doc.youzi177.cn;
    return 301 https://$host$request_uri;
}
server {
    listen 443 ssl http2;
    server_name doc.youzi177.cn;
    ssl_certificate     /home/keys/cert.pem;
    ssl_certificate_key /home/keys/key.pem;
    ssl_dhparam         /home/keys/dhparam.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    location / {
        proxy_pass http://show-server;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

- `server showdoc:80`其实就是showdoc这个容器，因为nginx和showdoc是在一个网段里面的，所以可以相互通信
- server_name 后面记得改回自己域名
- 重新启动`nginx`：`docker restart some-nginx`
- 最后在`DNS`服务商那里把域名解析到服务器IP，直接访问域名就可以看到showdoc这个网站，并且是`https`的
