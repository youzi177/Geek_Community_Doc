# 集成数据库

本章主要两个内容，一个是使用`docker compose`安装`mongdb`和`redis`，并进行基础的账户配置，下载可视化客户端，第二个是后端集成数据库，简单的CURD处理。

## docker安装mongo

在home目录下创建mongo文件夹，把以下yml文件上传安装

```yml
services:
  mongo:
    image: mongo
    restart: always
    container_name: 'mongo-youzi177'
    environment:
      MONGO_INITDB_ROOT_USERNAME: root
      MONGO_INITDB_ROOT_PASSWORD: aa123456
    ports:
      - 1111:27017
    volumes: 
      - /home/mongo/mongodb:/data/db
networks:
  default:
    name: https
    external: true
```

安装：

```
docker compose up -d
```

这里的第一个端口是实际需要访问的端口，也就是会占用服务器端口，请确保服务器这个端口是开放的没有其他应用，并且也要注意服务器商的这个端口要开放

container_name是自定义这个应用名称，`MONGO_INITDB_ROOT_USERNAME`可以改为名字，`MONGO_INITDB_ROOT_PASSWORD`改密码，建议复杂一点

测试：

```
docker ps
```

看到运行容器就没问题了

## mongo创建业务数据库

在之前安装的时候，有一个账户和密码，但是这个root级别的，权限太高，一般不会直接使用，这里创建一个用户和数据库。

1. 进入容器：

   ```shell
   docker exec -it mongo-youzi177 mongosh
   ```

   这里`mongo-youzi177` 是自己自定义的容器名称

2. 切换到 admin 库

   ```shell
   use admin
   ```

3. 登录管理员账号

   ```shell
   db.auth("root","你的管理员密码")
   ```

   成功：1

4. 查看数据库：

   ```
   show dbs
   ```

5. 创建业务数据库

   ```shell
   use geekdb
   ```

6. 创建业务用户

   ```shell
   db.createUser({
       user: "youzi177",
       pwd: "设置一个强密码",
       roles: [
           {
               role: "readWrite",
               db: "geekdb"
           }
       ]
   })
   ```

7. 测试业务用户登录：

   退出：

   ```
   exit
   ```

   重新进入

   ```
   docker exec -it mongo-youzi177 mongosh
   ```

   登录业务库：

   ```
   use geekdb
   ```

   认证：

   ```
   db.auth("youzi177","你的密码")
   ```

   返回：1，说明成功。

## 下载安装使用Studio 3T 

官方下载地址:https://robomongo.org/download.php

让你提供一个邮箱，然后下载链接发到邮箱上，邮箱收到就可以点击下载了，这个是社区版本，没有30天期限的，下载安装之后就可以使用

使用：

- 主界面点击：Open Connection Manager
- 点击：New connection
- 选择：Manually configure my connection settings
- Connection name填这个连接的名称，这里跟数据库无关，可以自定义
- Server和Prot这里就是写自己的服务器IP和端口（记得开放这个端口）
- 先别save，上一步是创建服务器的基础配置，这一步才是配置数据库：切换到右边tap->Authentication，Authentication Mode下拉选择Basic (SCRAM-SHA-256)，User name和Password就填写数据库的用户名和密码，记住是数据库的，Authentication DB填写数据库名称，就是上面创建的geekdb，点击左下的Test Connection可以测试连接是否成功，最后点击save就可以了
- 至此，服务器搭建`mongo`完成

## docker安装redis

在home目录下创建`redis`文件夹，把以下yml文件上传安装

```yml
services: 
  redis:
    image: 'redis'
    restart: always
    container_name: 'redis-youzi177'
    ports: 
      - 15001:6379
    volumes: 
      - /home/redis/redisdata:/data
    command: ["redis-server","--requirepass","1111"]

networks:
  default:
    name: https
    external: true
```

安装：

```
docker compose up -d
```

这里的第一个端口是实际需要访问的端口，也就是会占用服务器端口，请确保服务器这个端口是开放的没有其他应用，并且也要注意服务器商的这个端口要开放

container_name是自定义这个应用名称，`command`里面的`--requirepass`是密码，建议复杂一点

测试：

```
docker ps
```

看到运行容器就没问题了

## 后端集成mongo

这里使用到的一个包：mongoose，操作 MongoDB 的一个 ODM（Object Data Modeling，对象数据建模）库

安装 Mongoose

```shell
npm install mongoose
```

新建文件：src\config\index.js

```js
import 'dotenv/config';

const DB_URL = process.env.DB_URL;
export default { DB_URL };
```

.env

```
DB_URL=mongodb://数据库用户名:密码@ip:端口/数据库名称
```

注意：这里的密码如果存在特殊字符。需要特殊处理一下

| 字符 | 编码  |
| :--: | :---: |
| `@`  | `%40` |
| `:`  | `%3A` |
| `/`  | `%2F` |
| `?`  | `%3F` |
| `#`  | `%23` |
| `%`  | `%25` |
| 空格 | `%20` |

创建:src\config\DBHelpler.js

```js
import mongoose from 'mongoose';
import config from './index';

mongoose.connect(config.DB_URL).catch((err) => {
  console.error('MongoDB initial connection error:', err);
});
console.log('当前DB_URL:', config.DB_URL);
// 连接成功
mongoose.connection.on('connected', () => {
  console.log(`Mongoose connected to ${config.DB_URL}`);
});
// 连接异常
mongoose.connection.on('error', (err) => {
  console.error('Mongoose connection error:', err);
});

// 断开连接
mongoose.connection.on('disconnected', () => {
  console.log('Mongoose disconnected');
});

export default mongoose;

```

创建src\model\User.js

```js
import mongoose from '../config/DBHelpler';
const Schema = mongoose.Schema;
const UserSchema = new Schema({
  usename: { type: String },
  password: { type: String },
  name: { type: String },
});
const UserModel = mongoose.model('users', UserSchema);

export default UserModel;

```

添加注册接口的业务逻辑：src\api\loginController.js

```js
async reg(ctx) {
    // 接受用户数据
    const { body } = ctx.request;
    //此处省略很多判断逻辑,测试数据库注册逻辑
    const user = new User({
        username: body.username,
        name: body.name,
        password: body.password,
    });
    //写库
    const result = await user.save();
    ctx.body = {
        code: 200,
        data: result,
        msg: '注册成功',
    };
}
}
```

添加路由：src\router\modules\loginRouter.js

```js
//注册
router.post('/reg', loginController.reg);
```

简单说下这些代码干嘛：

1. `env`文件，添加数据库的连接，`DB_URL=mongodb://数据库用户名:密码@ip:端口/数据库名称`
2. 创建index文件，存放配置文件，这个文件可能现在简单，感觉没有存在必要，但是后期业务复杂会有用处
3. `DBHelpler`创建数据库连接，监听成功、异常和断开的方法
4. 创建User，这个是数据库表
5. `reg`这个注册接口是对前端传回来的数据，然后接收写入数据库，返回code200
6. `loginRouter`添加reg这个方法

测试接口：


发送数据结构；

```json

```

```js
const { createClient } = require('redis');
const config = require('./index');
const client = createClient({
  socket: {
    host: config.REDIS.host,
    port: config.REDIS.port,
    // 重连策略
    reconnectStrategy: (retries, cause) => {
      if (cause.code === 'ECONNREFUSED') {
        return new Error('Redis server refused connection');
      }
      if (retries > 10) {
        return new Error('Redis retry exhausted');
      }
      // 重连间隔
      return Math.min(retries * 100, 3000);
    },
  },
  password: config.REDIS.password || undefined,
});

client.on('connect', () => {
  console.log('Redis connecting...');
});

client.on('ready', () => {
  console.log('Redis ready');
});

client.on('error', (err) => {
  console.error('Redis Error:', err);
});

client.on('reconnecting', () => {
  console.log('Redis reconnecting');
});
// 设置值
const setValue = async (key, value, time) => {
  //如果是空的，就直接返回，不存
  if (typeof value === 'undefined' || value === null || value === '') {
    return;
  }
  //如果是一个字符串，那就判断time是否存在，存在就设置过期时间，没有就直接set
  if (typeof value === 'string') {
    if (time) {
      client.set(key, value, {
        EX: time,
      });
    } else {
      client.set(key, value);
    }
  }
  //如果是一个对象，使用hSet，然后判断有没有设置过期时间，有的话再设置过期时间
  if (typeof value === 'object') {
    client.hSet(key, value);
    if (time) {
      await client.expire(key, time);
    }
  }
};

// 获取值
const getValue = async (key) => {
  //client里面取这个type
  const type = await client.type(key);
  //判断是hash还是string，这两个的取值方式不一样
  if (type === 'hash') {
    return client.hGetAll(key);
  }
  if (type === 'string') {
    return client.get(key);
  }
};
async function main() {
  await client.connect();
  await setValue('key', { name: '小封鸭', age: '18' }, 20);
  const result = await getValue('key');
  console.log(result);
}

main();

```

来看看这段代码在干嘛：

- 首先引入配置文件congig和redis
- createClient创建redis客户端，然后配置socket，重连策略
- 监听redis的connect、ready、error和reconnecting事件
- setValue和getValue，这里都做了逻辑判断
- 最后一个测试main方法来测试redis是否正常
- 运行：`node src\config\RedisConfig.js`
