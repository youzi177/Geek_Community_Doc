# 优化邮箱服务环境变量

在之前的邮箱服务配置中，有以下代码：

```js
// 邮箱服务基本配置
const transporter = nodemailer.createTransport({
    host: 'smtp.126.com',
    port: 465, //一般是465 SSL
    secure: true, //465端口需要修改为true  use STARTTLS (upgrade connection to TLS after connecting)
    auth: {
        user: 'XXX@126.com',
        pass: 'you-password',
    },
});
```

这是非常敏感的数据，如果提交之前改为xxx，开发的时候又改回正确，这非常麻烦，还有不小心就push到仓库的风险，所以这里肯定不能这样写了

## 使用dotenv

这里使用dotenv来解决上边的问题：

安装:

```shell
npm install dotenv
```

创建 `.env`

```
SMTP_USER=你真实的邮箱
SMTP_PASS=你真实的密码
```

注意这里不要加引号，直接写值就可以。

在 `.gitignore` 中添加：

```
.env
.env.local
.env.production
```

使用：

```js
import 'dotenv/config';
  // 邮箱服务基本配置
  const transporter = nodemailer.createTransport({
    host: 'smtp.126.com',
    port: 465, //一般是465 SSL
    secure: true, //465端口需要修改为true  use STARTTLS (upgrade connection to TLS after connecting)
    auth: {
      user: process.env.SMTP_USER,
      pass: process.env.SMTP_PASS,
    },
  });
```

## nodemailer的注意点

在发送邮箱from这里，需要写上你真实的邮箱，否则接口会报错

```js
from: `"Geek极客社区" ${process.env.SMTP_USER}`, // sender address
```
