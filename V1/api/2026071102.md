# 配置邮箱服务接口

使用`Nodemailer`实现找回密码时候的邮箱发送问题，为了演示，这里使用的126邮箱，但是需要注意，这种邮箱限额非常少，大概一天几十一百封，再多可能封号或者别人收不到验证码，如果想稳定可以使用云平台的邮箱服务，额度高，也有免费额度，稳定性也高，这里只是为了验证可行性，后续项目上线时再更换邮箱服务。

本章内容主要是开发一个接口，但是远远没有可用的阶段，之后还有修改邮箱账号，注册时候的验证码这两个功能都需要，但是这涉及到数据库和用户信息，在后续开发首页模块后期会完善这些功能，现在阶段就是简单一个接口。

## 开启邮箱授权码

打开登录126邮箱，选择设置，找到**POP3/SMTP/IMAP**，开启POP3/SMTP服务，保存密钥。

126邮箱的服务器地址配置，等会需要

```
服务器地址：
POP3服务器: pop.126.com
SMTP服务器: smtp.126.com
IMAP服务器: imap.126.com
```

## 安装使用Nodemailer

官方文档：https://nodemailer.com/

安装：

```shell
npm install nodemailer
```

使用：在src下创建config，在config下创建MailConfig.js文件

```js
//src\config\MailConfig.js
const nodemailer = require('nodemailer');

async function main() {
  // 邮箱服务基本配置
  const transporter = nodemailer.createTransport({
    host: 'smtp.126.com',
    port: 465, //一般是465 SSL
    secure: true, //465端口需要修改为true  use STARTTLS (upgrade connection to TLS after connecting)
    auth: {
      user: 'XXXXX@126.com',
      pass: 'you-password',
    },
  });
  //发送邮件
  const info = await transporter.sendMail({
    from: '"认证邮件" <XXXXX@126.com>', // sender address
    to: 'XXXX@qq.com', // 发给谁，可以是一个列表
    subject: 'Hello', // subject line
    text: 'Hello world?', // plain text body
    html: '<b>Hello world?1</b>', // HTML body
  });
  console.log('发送成功');
  console.log('MessageId:', info.messageId);
  console.log('Response:', info.response);
}
//捕获异常
main().catch(console.error);

```

运行：

```shell
node src/config/MailConfig.js
```

如果能看到打印信息说明发送成功，可以到接受的邮箱看是否收到邮件

```js
console.log('MessageId:', info.messageId);
console.log('Response:', info.response);
```

## 开发邮箱服务的接口

邮箱服务配置完成之后 ，接下来就需要对接口进行开发，这里先对配置文件改写一下：

```js
import nodemailer from 'nodemailer';
async function send(sendInfo){
      return ('MessageId:', info.messageId);
}
export default send;
```

- 改为import；
- 函数名字改为send，参数`sendInfo`，之前函数里面的`sendInfo`就删除掉
-  `return ('MessageId:', info.messageId);`返回`info.messageId`
- 最后`export default send`，就不要执行了，导出函数

创建`src\api\loginController.js`

```js
import send from '../config/MailConfig';
import moment from 'moment';
class loginController {
  constructor() {}
  async forget(ctx) {
    const { body } = ctx.request;
    try {
      //这里的body.username应该查询数据库然后再发送邮件
      const result = await send({
        code: '1234',
        expire: moment().add(30, 'minutes').format('YYYY-MM-DD HH:mm:ss'),
        email: body.username,
        user: 'Geek极客',
      });
      ctx.body = {
        code: 200,
        data: result,
        msg: '邮件发送成功',
      };
    } catch (error) {
      console.log(error);
    }
  }
}
export default new loginController();

```

- 导入邮箱服务的配置，刚才sendInfo是参数，在这里就需要根据前端的信息进行传参
- expire这里使用moment这个库进行格式化，安装：`npm install moment --save`
- email：前端输入传回的email，一般这里需要先到数据库查询有没有这个邮箱，再决定是否发送，这里没有数据库，就先写死，后期完善
- code和user都是一样的，也是需要数据库才可以，这里都先写死固定

创建`src\router\modules\loginRouter.js`

```js
import Router from 'koa-router';
import loginController from '../../api/loginController.js';
const router = new Router();
router.prefix('/login');
// 找回密码
router.post('/forget', loginController.forget);

export default router;

```

引入

```js
//src\router\routes.js
import combineRoutes from 'koa-combine-routers';
import publicRouter from './modules/publicRouter.js';
import loginRouter from './modules/loginRouter.js';
// 合并路由
export default combineRoutes(publicRouter, loginRouter);

```

运行项目：`npm run dev`
